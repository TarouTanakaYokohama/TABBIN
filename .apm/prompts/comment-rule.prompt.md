# Comment Review

現在の変更差分に含まれるコメントを確認してください。

## 方針

- 自明な処理を説明するだけのコメントは不要です。
- 複雑な条件、非直感的な制約、外部仕様への依存を補うコメントは残します。
- コメントよりも命名や構造で意図を表せる場合は、コード側の改善を提案します。
- 日本語と英語が混在する場合は、周辺ファイルの慣習に合わせます。

## 出力

削除または修正すべきコメントがあれば、理由と具体的な修正案を示してください。
問題がなければ、その旨を短く答えてください。
